# AGENTS.md

Sparrow is a Go CLI for infrastructure monitoring (health, latency,
DNS, traceroute checks). Single module, no workspace.

## Commands

```bash
# Unit tests (matches CI exactly)
go test --race --count=1 -v ./...

# Single package
go test --race --count=1 -v ./pkg/checks/dns/...

# Lint (requires golangci-lint)
golangci-lint run --config .golangci.yaml --fix

# Regenerate mocks (moq via go tool)
go generate ./...

# Format
gofumpt -l -w .
```

## Code generation

Mocks use `go tool moq` (declared as a tool dependency in `go.mod`).
Generated files end in `_moq.go`. Run `go generate ./...` after
changing interfaces in `pkg/api`, `pkg/checks`, `pkg/config`,
`pkg/checks/dns`, or `pkg/sparrow/metrics`.

Pre-commit runs `go generate ./...` automatically — if you skip
hooks, regenerate manually.

## Project layout

- `cmd/` — Cobra CLI (root, run, flags)
- `pkg/checks/` — check implementations (health, latency, dns,
  traceroute) each in their own subpackage
- `pkg/api/` — HTTP API (chi router)
- `pkg/config/` — config loading (Viper)
- `pkg/sparrow/` — orchestrator wiring checks together
- `pkg/db/` — in-memory results store
- `internal/logger/` — structured logging helpers
- `chart/` — Helm chart; docs auto-generated by `helm-docs`
- `e2e/` — end-to-end tests (traceroute only, needs infra)

## Conventions

- Every new `.go` file must start with this exact SPDX header
  (CI runs REUSE compliance and will fail without it):

  ```go
  // SPDX-FileCopyrightText: 2026 Deutsche Telekom IT GmbH
  //
  // SPDX-License-Identifier: Apache-2.0
  ```
- Lint config is strict: `funlen` (50 statements), `gocyclo` (15),
  `dupl` (100 threshold), `mnd` (no magic numbers except 0-3).
  Test files are exempt from `mnd`, `revive`, and `goconst`.
- Use `gofumpt` (not `gofmt`) for formatting.
- Version is injected via `-ldflags "-X main.version=..."` at build
  time (goreleaser handles this).
